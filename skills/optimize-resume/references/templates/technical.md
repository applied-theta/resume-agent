# Technical Resume Template

This template is used for candidates targeting technical roles where depth of technical expertise is the primary differentiator. It is preferred for software engineering, DevOps, data engineering, SRE, and similar roles regardless of archetype, unless the candidate is executive-level or a career changer.

## Key Differences from Standard Template

### Enhanced Technical Skills Section

The technical skills section is more detailed and categorized than the standard template:
- Explicit categories: Languages, Frameworks, Cloud, Databases, Tools, Methodologies
- Include both acronyms and full terms for each technology
- Order categories by relevance to the target role
- 15-25 skills total across all categories

### Projects and Open Source Sections

Technical resumes include dedicated sections for:
- **Projects:** Personal or side projects demonstrating initiative and depth beyond work experience
- **Open Source Contributions:** Shows community engagement and public code quality
- These sections are particularly important for entry-level and mid-level candidates

### GitHub/Portfolio Links

- Include GitHub profile link in contact information
- Link to specific project repos in the Projects section
- Include portfolio/personal site if it showcases technical work

### Technical Depth in Bullets

Experience bullets focus on technical specifics:
- Name the specific technologies, architectures, and patterns used
- Include system scale metrics (requests/second, data volume, uptime)
- Describe technical decisions and their outcomes
- Reference specific technical methodologies (TDD, event-driven architecture, etc.)

## Guidance by Experience Level

### Junior / Entry-Level (0-2 Years)

1. Contact Information (include GitHub)
2. Summary / Objective
3. Education (prominent: GPA if > 3.0, relevant coursework)
4. Projects (2-3 substantial projects with technical details)
5. Experience (internships)
6. Technical Skills
7. Open Source / Hackathons (if applicable)

**Emphasis:** Projects are the primary evidence of technical ability. Include architecture decisions and technical challenges solved.
**De-emphasis:** Limited experience is expected. Focus quality over quantity.
**Length:** 1 page strictly.

### Mid-Level (3-7 Years)

1. Contact Information (include GitHub)
2. Professional Summary
3. Technical Skills (detailed categories)
4. Professional Experience (3-5 bullets per role showing increasing scope)
5. Projects (1-2 notable projects)
6. Education
7. Certifications

**Emphasis:** Professional experience is the centerpiece. Show progression from implementer to technical decision-maker.
**De-emphasis:** Personal projects become supplementary. Education is brief.
**Length:** 1-2 pages.

### Senior / Staff / Principal (8+ Years)

1. Contact Information
2. Professional Summary (emphasize architecture and technical leadership)
3. Technical Skills / Areas of Expertise
4. Professional Experience (4-6 bullets for recent roles, emphasize technical decisions with business impact)
5. Speaking / Publications (if applicable)
6. Education
7. Certifications

**Emphasis:** Architectural decisions, system design, technical strategy, and mentorship. Show influence beyond your own code.
**De-emphasis:** Personal projects unless they are significant open source. Basic technologies assumed.
**Length:** 2 pages acceptable.

## Technical Bullet Writing

### System Performance Metrics

```
Architected event-driven data pipeline processing [X]M events/day with
sub-[X]ms p99 latency, replacing batch ETL system that required [X]-hour
processing windows
```

### Scale and Reliability

```
Designed and deployed microservices architecture serving [X]K requests/second
across [X] services with 99.[X]% uptime, handling [X]x traffic growth without
infrastructure cost increase
```

### Infrastructure and DevOps

```
Implemented infrastructure-as-code using Terraform, reducing environment
provisioning time from [X] days to [X] minutes and eliminating configuration
drift across [X] environments
```

### Data and Machine Learning

```
Built real-time recommendation engine processing [X]M user interactions daily,
improving click-through rate by [X]% and generating $[X]M in incremental
annual revenue
```

---

## What to Emphasize

- **System scale:** Requests/second, data volume, user count, uptime SLAs
- **Performance improvements:** Latency reductions, throughput increases, cost savings
- **Architecture decisions:** Why you chose a particular approach, not just what you built
- **Technical leadership:** Design reviews, tech talks, mentoring, RFC processes
- **Production ownership:** On-call experience, incident response, reliability improvements
- **Quantified outcomes:** Every technical achievement should have a measurable result

## What to De-Emphasize

- **Soft skills in bullets:** Save for the summary; bullets should be technically specific
- **Meeting attendance:** "Attended standups" is not an achievement
- **Basic tool usage:** "Used Git for version control" is assumed for any developer
- **Unquantified claims:** "Improved performance" without metrics is weak
- **Obsolete technologies:** Remove technologies that are no longer relevant to your target roles

---

## Formatting Guidelines

- **Font:** Clean, modern (Calibri, Arial, Consolas for code snippets if needed)
- **Margins:** 0.5-1 inch on all sides
- **Font size:** 10-11pt for body text (technical resumes can use slightly smaller font to fit more content)
- **Code/tech formatting:** Technology names in plain text, not code blocks, for ATS compatibility
- **Links:** Include GitHub and portfolio URLs; ensure they are also visible as plain text
- **No tables, text boxes, columns, or images** -- ATS compatibility required
- **Length:** 1 page for entry-level, 1-2 pages for mid-level, 2 pages for senior
